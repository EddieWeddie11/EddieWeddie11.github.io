What are the best practices associated with using classes vs. ids?

Class

In CSS, a class groups elements that are the same or similar. You can include any number of elements in a class, and each element can belong to multiple classes. Each class has specific CSS attributes, such as color and font-size.

CSS classes are like a real-life class of students who share similarities in age, area codes, interests, or life goals.

When inserting a class syntax, ensure the name you give the class between quotation marks (“”) is distinct, the name you give it can be literally anything.


Syntax for class in the HTML

h1 class= "example 1"

h2 class= "example 2"

Syntax for class in the CSS Selector file (Use '.' (period symbol) for identifying your class))

.example {color: blue; font-size: 14px;}


IDs

An ID uniquely identifies a single HTML tag, and each tag can only have one ID. Every ID has specific CSS attributes that apply only to that element.

In a real-life analogy, while students may share attributes, each has a unique IRD Number or passport number that identifies them individually.

When inserting a class syntax, ensure the name you give the class between quotation marks (“”) is distinct, the name you give it can be literally anything.


Syntax for ID in the HTML

h1 id= ”unique 1”

h2 id= ”unique 2”

Syntax for ID in the CSS Selector file (Use ‘#’ for identifying your ID)

#unique {color: red; font-size: 16px;}


Best practices

  • Use an ID when you need to style one specific, unique element on a page
  • Use a class when you need to style multiple elements with the same or similar styling
  • IDs are for single, unique items, while classes are for groups of items that share styles
  • Keep IDs distinct to avoid any conflicts and ensure clear, maintainable code
